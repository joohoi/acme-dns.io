Security

Providing a limited API key, that only provides access to change a single TXT record; the magic name used by ACME DNS challenge: _acme-challenge.yourdomain.tld

Accessibility

Enabling you to use ACME DNS challenge even if your current DNS server software or service doesn't provide and automatable API for updating it.

How it works?

ACME-DNS creates a limited DNS server that serves TXT records used to carry ACME challenge tokens when using the DNS challenge.

Get an API key and a unique subdomain from ACME-DNS service. Add a CNAME record to your zone for the magic subdomain of the domain you want to automate DNS challenge for (eg. _acme-challenge.yourdomain.tld) pointing to the unique ACME-DNS subdomain you have. After this you are able to use your ACME-DNS API keys for the challenge automation.

Self-hosted

You are encouraged to run acme-dns as a self-hosted service. Go get the code from Github. Acme-dns is written in go, and runs on wide variety of operating systems.
